# LL Sims 4 Auto Updater

Локальный проект для отслеживания обновлений LoversLab-модов, безопасной очереди скачивания и авто-деплоя в папку `Mods`.

## Ключевые возможности

- UI со списком модов и чекбоксом `Auto` (включен в автоотслеживание и автоочередь)
- Автопоиск модов из категории `The Sims 4` + добавление по прямому URL
- Проверка версий по `softwareVersion` / `dateModified`
- Обновления ставятся в **персистентную очередь** (`data/queue.json`)
- Очередь обрабатывается фоново, может идти **несколько дней подряд**
- Деплой файлов в `Mods` через `copy` / `hardlink` / `symlink`
- Сохранение состояния рантайма и лимитов (`data/runtime.json`)

## Анти-бан архитектура (встроено)

В проекте уже реализованы защитные механизмы:

- строго последовательные скачивания (без параллельных download-потоков)
- случайная пауза между скачиваниями (`download_min_delay_seconds`..`download_max_delay_seconds`)
- лимит на час / сутки (`max_downloads_per_hour`, `max_downloads_per_day`)
- экспоненциальный backoff (`backoff_base_minutes`, `backoff_max_minutes`)
- глобальные cooldown-режимы по сигналам `429`, `503`, Cloudflare challenge/deny
- сохранение `not_before` для каждого queue-item (без потери прогресса при перезапуске)

## Что показал ресерч

Официальных публичных числовых квот (X загрузок/час) на LL в открытых правилах не найдено, но есть косвенные/сообщественные сигналы:

- обсуждение задержки и ограничений на скачивание:  
  `https://www.loverslab.com/topic/78741-download-delay-limitations/`
- обсуждение 503 и срабатывания защиты при агрессивных внешних загрузчиках:  
  `https://www.loverslab.com/topic/170307-downloading-mods-using-wabbajack-503-for-every-file/`
- базовые правила/terms (без конкретных quota-чисел):  
  `https://www.loverslab.com/guidelines/`  
  `https://www.loverslab.com/terms/`

Из-за этого в приложении заложены **консервативные дефолты**.

## Логин/пароль

Пароль не хранится и не требуется в приложении.

Нужна активная сессия LL: вставь cookie из залогиненного браузера в поле `Cookie`.
Если cookie устарел, вместо файла можно получить страницу логина/Cloudflare.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск

```bash
python app.py
```

Открой: `http://127.0.0.1:8765`

## Быстрый сценарий

1. Укажи `Папка Mods` и метод деплоя
2. Вставь `Cookie`, нажми `Сохранить настройки`
3. `Обновить список модов`
4. Включи `Auto` у нужных модов
5. Нажми `Поставить обновления в очередь`
6. Дай воркеру обрабатывать очередь (или кнопкой `Обработать 1 элемент очереди`)

## Ограничения

- `.rar`/`.7z` требуют установленный 7-Zip (`7z.exe`)
- `symlink` на Windows может требовать Developer Mode / повышенные права
- `hardlink` работает только в пределах одного тома
- Нулевого риска бана не существует; есть только снижение риска через троттлинг
